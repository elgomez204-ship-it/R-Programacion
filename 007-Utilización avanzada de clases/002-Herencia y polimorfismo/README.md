# Herencia y Polimorfismo

## Introducción

La herencia es un mecanismo fundamental de la programación orientada a objetos que permite que una clase herede atributos y métodos de otra. El polimorfismo permite que objetos de diferentes clases respondan al mismo mensaje de forma diferente. Juntos, estos conceptos son la base para crear jerarquías de clases flexibles y reutilizables.

## Aspectos Técnicos

- **Herencia Simple**: Una clase hereda de una única clase padre.
- **Herencia Múltiple**: Una clase hereda de múltiples clases.
- **Clase Base y Derivada**: Relación entre clase padre e hijo.
- **Polimorfismo Dinámico**: Métodos que se resuelven en tiempo de ejecución según el tipo real del objeto.
- **Sobreescritura de Métodos**: Reemplazo de métodos heredados con nuevas implementaciones.
- **Métodos y Atributos Heredados**: Acceso a funcionalidades de la clase padre.
- **Super()**: Llamada a métodos de la clase padre desde la clase derivada.

## Para qué Sirve

- Crear jerarquías de clases que modelan relaciones "es un" en el dominio del problema.
- Reutilizar código común en una clase base evitando duplicación.
- Facilitar extensión de funcionalidad mediante herencia.
- Permitir que diferentes clases tengan interfaz común pero comportamiento diferente.
- Mejorar la abstracción y el diseño general del sistema.
- Aplicar el principio de sustitución de Liskov para código robusto.

## Conclusión

La herencia y el polimorfismo son pilares del diseño orientado a objetos que habilitan la creación de sistemas extensibles y mantenibles. Dominar estos conceptos permite escribir código más limpio, reutilizable y profesional que escala mejor con el crecimiento de la complejidad.
