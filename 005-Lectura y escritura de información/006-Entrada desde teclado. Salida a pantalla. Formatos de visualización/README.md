# Entrada desde teclado. Salida a pantalla. Formatos de visualización

## Introducción

La interacción con el usuario a través de teclado e pantalla es el mecanismo básico de comunicación en aplicaciones de línea de comandos. La entrada permite recibir información del usuario, mientras que la salida formateada permite presentar resultados de manera clara y legible. El control del formato de visualización es crucial para crear interfaces de usuario efectivas.

## Aspectos Técnicos

- **Entrada desde teclado**: Lectura de datos proporcionados por el usuario (líneas, caracteres, números).
- **Entrada estándar (stdin)**: Flujo de entrada por defecto del programa.
- **Salida a pantalla**: Escritura de información en la consola o terminal.
- **Salida estándar (stdout)**: Flujo de salida normal del programa.
- **Salida de error (stderr)**: Flujo separado para mensajes de error.
- **Formatos de visualización**: Control de precisión decimal, ancho de campo, alineación, etc.
- **Cadenas formateadas**: Uso de caracteres especiales, tabulaciones, saltos de línea.

## Para Qué Sirve

La entrada/salida y formatos de visualización sirven para:

1. **Interactuar con usuarios**: Permitir que los usuarios proporcionen datos al programa.
2. **Mostrar resultados**: Presentar información de manera comprensible.
3. **Mejorar legibilidad**: Usar formatos y espaciado para hacer datos más claros.
4. **Validar entrada**: Solicitar al usuario datos válidos en el formato correcto.
5. **Crear interfaces**: Construir interfaces de usuario intuitivas en línea de comandos.

## Conclusión

Dominar la entrada y salida de datos con formatos apropiados es fundamental para crear aplicaciones interactivas y fáciles de usar. La capacidad de presentar información de manera clara y solicitar entrada del usuario de forma intuitiva es lo que distingue aplicaciones profesionales de proyectos básicos.
